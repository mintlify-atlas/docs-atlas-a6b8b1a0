## Overview

The Click Creators Scraper Client supports simultaneous scraping campaigns across four major social media platforms: Instagram, TikTok, Threads, and X (formerly Twitter). Each platform operates independently with isolated data and dedicated workflows.

<Note>
  Multi-platform scraping allows you to:
  - Track the same influencer across different platforms
  - Compare audience engagement across social networks
  - Maximize reach by targeting followers on multiple channels
  - Maintain separate campaigns with isolated data per platform
</Note>

## Multi-Platform Architecture

### Data Isolation

Each platform job maintains complete data isolation:

```typescript
// Platform-specific job structure
interface ScrapingJob {
  job_id: string           // Unique per job
  platform: Platform       // 'instagram' | 'tiktok' | 'threads' | 'x'
  base_id: string          // Airtable base for isolation
  influencer_name: string  // Same influencer, different platforms
}
```

**Example: Same Influencer, Multiple Platforms**

```typescript
// Andrew Tate across platforms
{
  job_id: 'uuid-1',
  platform: 'instagram',
  influencer_name: 'Andrew Tate',
  base_id: 'appInstagramBase123'
}

{
  job_id: 'uuid-2', 
  platform: 'tiktok',
  influencer_name: 'Andrew Tate',
  base_id: 'appTikTokBase456'
}

{
  job_id: 'uuid-3',
  platform: 'threads',
  influencer_name: 'Andrew Tate',
  base_id: 'appThreadsBase789'
}
```

<Warning>
  Each platform requires its own Airtable base. Never share bases across platforms, even for the same influencer.
</Warning>

## Creating Multi-Platform Jobs

### Strategy 1: Same Influencer, Multiple Platforms

<Steps>
  <Step title="Create Instagram Job">
    Navigate to `/configure` and create the first job:
    
    ```
    Influencer Name: Andrew Tate
    Platform: Instagram
    Number of VAs: 80
    Airtable Base: "Andrew Tate's Instagram Campaign"
    ```
  </Step>

  <Step title="Create TikTok Job">
    Return to `/configure` and create the second job:
    
    ```
    Influencer Name: Andrew Tate
    Platform: TikTok  
    Number of VAs: 60
    Airtable Base: "Andrew Tate's TikTok Campaign"
    ```
    
    <Note>
      Notice the different VA count. Adjust based on platform-specific follower volume.
    </Note>
  </Step>

  <Step title="Create Threads Job">
    Create the third job:
    
    ```
    Influencer Name: Andrew Tate
    Platform: Threads
    Number of VAs: 40
    Airtable Base: "Andrew Tate's Threads Campaign"
    ```
  </Step>

  <Step title="Create X Job">
    Create the final job:
    
    ```
    Influencer Name: Andrew Tate
    Platform: X
    Number of VAs: 80
    Airtable Base: "Andrew Tate's X Campaign"
    ```
  </Step>
</Steps>

**Result:** Four independent campaigns for the same influencer, each with:
- Separate profile pools
- Independent VA assignments
- Isolated statistics
- Platform-specific Airtable bases

<Accordion title="Campaign Success Rate">
  Monitor campaign completion across platforms:
  
  ```sql
  -- Success rate by platform
  SELECT 
    sj.platform,
    COUNT(c.campaign_id) as total_campaigns,
    SUM(CASE WHEN c.status = true THEN 1 ELSE 0 END) as successful,
    (SUM(CASE WHEN c.status = true THEN 1 ELSE 0 END)::float / 
     COUNT(c.campaign_id) * 100) as success_rate
  FROM campaigns c
  JOIN scraping_jobs sj ON c.base_id = sj.base_id
  GROUP BY sj.platform
  ```
</Accordion>

## Advanced Multi-Platform Strategies

### Cross-Platform Audience Analysis

<Accordion title="Identify Overlap">
  Find users active on multiple platforms:
  
  ```sql
  -- Find usernames appearing on multiple platforms
  SELECT 
    gu1.username,
    COUNT(DISTINCT sj.platform) as platform_count,
    ARRAY_AGG(DISTINCT sj.platform) as platforms
  FROM global_usernames gu1
  JOIN scraping_jobs sj ON gu1.base_id = sj.base_id
  GROUP BY gu1.username
  HAVING COUNT(DISTINCT sj.platform) > 1
  ```
  
  Use this data to:
  - Prioritize multi-platform users
  - Cross-promote content
  - Identify superfans
</Accordion>

<Accordion title="Platform Migration Tracking">
  Track users moving between platforms:
  
  ```typescript
  // Identify users who joined new platform
  const threadsUsers = await getJobUsernames(threadsJobId)
  const instagramUsers = await getJobUsernames(instagramJobId)
  
  const newToThreads = threadsUsers.filter(tu => 
    !instagramUsers.find(iu => iu.username === tu.username)
  )
  
  console.log(`${newToThreads.length} users are new to Threads`)
  ```
</Accordion>

## Troubleshooting Multi-Platform Issues

### Platform-Specific Errors

<Accordion title="Different Scrape Volumes">
  **Issue:** Instagram returns 10K profiles, TikTok returns 50K profiles
  
  **Solution:**
  - This is normal - platforms have different scraping yields
  - Adjust daily targets per platform:
    ```bash
    # Platform-specific targets (future enhancement)
    INSTAGRAM_DAILY_TARGET=14400
    TIKTOK_DAILY_TARGET=20000
    ```
</Accordion>

<Accordion title="Airtable Base Confusion">
  **Issue:** Profiles from one platform appearing in another
  
  **Solution:**
  1. Verify each job has unique `base_id`
  2. Check Airtable base URLs are different
  3. Query database:
     ```sql
     SELECT platform, base_id FROM scraping_jobs
     ```
  4. Ensure no shared base IDs across platforms
</Accordion>

<Accordion title="VA Assignment Errors">
  **Issue:** VAs receiving profiles from wrong platform
  
  **Solution:**
  1. Check campaign `base_id` matches job `base_id`
  2. Verify Airtable sync uses correct base
  3. Review daily_assignments table:
     ```sql
     SELECT da.*, sj.platform 
     FROM daily_assignments da
     JOIN scraping_jobs sj ON da.base_id = sj.base_id
     WHERE da.campaign_id = ?
     ```
</Accordion>
