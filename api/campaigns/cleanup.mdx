## Overview

The Cleanup endpoint removes campaigns that are 7+ days old, deletes their assignments, and marks their profiles as unused in the global pool. This maintains database efficiency and allows profiles to be reused in future campaigns.

## Endpoint

<ParamField path="method" type="string" default="POST">
  POST
</ParamField>

<ParamField path="endpoint" type="string">
  `/api/cleanup`
</ParamField>

## Request

No request body or parameters required. The endpoint automatically identifies and cleans up campaigns older than 7 days.

### Request Example

```bash
POST /api/cleanup
```

<Note>
This endpoint should be run as a scheduled cron job, not called manually from the UI.
</Note>

## Response

<ResponseField name="campaigns_cleaned" type="number" required>

</ResponseField>

## Cleanup Process

The cleanup follows these steps:

1. **Identify Old Campaigns** - Find campaigns where `campaign_date < NOW() - 7 days`
2. **Fetch Assigned Usernames** - Retrieve all usernames from `daily_assignments` for old campaigns
3. **Mark Profiles as Unused** - Set `used = false` in `global_usernames` table
4. **Delete Assignments** - Remove records from `daily_assignments` table
5. **Delete Campaigns** - Remove campaign records from `campaigns` table

```python
# Simplified cleanup logic
old_campaigns = get_campaigns_older_than(days=7)

for campaign in old_campaigns:
    # Get all assigned usernames
    usernames = get_assignments(campaign.campaign_id)
    
    # Mark as unused in global pool
    mark_usernames_unused(usernames)
    
    # Delete assignments
    delete_assignments(campaign.campaign_id)
    
    # Delete campaign
    delete_campaign(campaign.campaign_id)
```

## Campaign Lifecycle

Understanding the 7-day lifecycle:

| Day | Status | Action |
|-----|--------|--------|
| Day 0 | Created | Campaign assigned to VAs |
| Day 1-6 | Active | VAs working on profiles |
| Day 7 | Eligible | Campaign can be cleaned up |
| Day 8+ | Cleaned | Profiles freed, campaign deleted |

<Warning>
Profiles are permanently removed from campaigns after cleanup. Ensure VAs have completed their work before the 7-day window expires.
</Warning>

```python Python Scheduler
from apscheduler.schedulers.blocking import BlockingScheduler
import requests

    response = requests.post('http://localhost:5001/api/cleanup')

scheduler = BlockingScheduler()
scheduler.add_job(cleanup_campaigns, 'cron', hour=2, minute=0)
scheduler.start()
```

## Database Operations

The cleanup performs these SQL operations:

### 1. Find Old Campaigns

```sql
SELECT campaign_id, campaign_date, total_assigned
FROM campaigns
WHERE campaign_date < CURRENT_DATE - INTERVAL '7 days';
```

### 2. Get Assigned Usernames

```sql
SELECT DISTINCT username
FROM daily_assignments
WHERE campaign_id IN (
  SELECT campaign_id 
  FROM campaigns 
  WHERE campaign_date < CURRENT_DATE - INTERVAL '7 days'
);
```

### 3. Mark Profiles as Unused

```sql
UPDATE global_usernames
SET used = false
WHERE username IN (
  SELECT username 
  FROM daily_assignments 
  WHERE campaign_id = 'old-campaign-id'
);
```

### 4. Delete Assignments

```sql
DELETE FROM daily_assignments
WHERE campaign_id IN (
  SELECT campaign_id 
  FROM campaigns 
  WHERE campaign_date < CURRENT_DATE - INTERVAL '7 days'
);
```

### 5. Delete Campaigns

```sql
DELETE FROM campaigns
WHERE campaign_date < CURRENT_DATE - INTERVAL '7 days';
```

## Performance Considerations

<Note>
Cleanup operations are performed in transactions to ensure data consistency.
</Note>

**Typical cleanup metrics:**
- **Campaigns per cleanup:** 1-2 (running daily)
- **Profiles freed per campaign:** 14,400
- **Total operation time:** 5-15 seconds
- **Database rows deleted:** ~14,400-28,800 per run

**Optimization strategies:**
- Batch UPDATE operations (500 usernames at a time)
- Use database indexes on `campaign_date` and `campaign_id`
- Run during low-traffic hours (2 AM)
- Use VACUUM (PostgreSQL) after large deletions

## Monitoring and Logging

Recommended logging for cleanup operations:

```python
import logging

def cleanup_old_campaigns():
    logger = logging.getLogger('campaign_cleanup')
    
    try:
        old_campaigns = get_old_campaigns()
        logger.info(f"Found {len(old_campaigns)} campaigns to clean")
        
        total_freed = 0
        for campaign in old_campaigns:
            profiles = cleanup_campaign(campaign.id)
            total_freed += profiles
            logger.info(f"Cleaned campaign {campaign.id}: {profiles} profiles freed")
        
        logger.info(f"Cleanup complete: {len(old_campaigns)} campaigns, {total_freed} profiles")
        
        return {
            'success': True,
            'campaigns_cleaned': len(old_campaigns),
            'profiles_freed': total_freed
        }
        
    except Exception as e:
        logger.error(f"Cleanup failed: {str(e)}")
        return {'success': False, 'error': str(e)}
```

## Error Handling

| Error | Cause | Solution |
|-------|-------|----------|
| Database deadlock | Concurrent cleanup operations | Ensure only one cleanup runs at a time |
| Foreign key constraint | Orphaned assignments | Check database integrity before cleanup |
| Transaction timeout | Too many campaigns to clean | Reduce batch size or increase timeout |
| Partial cleanup | Process interrupted mid-operation | Use transactions to ensure atomicity |
