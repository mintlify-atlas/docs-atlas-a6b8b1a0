## Overview

The Airtable Sync endpoint pushes all 14,400 distributed profiles to their corresponding Airtable tables. This allows Virtual Assistants to access their assigned profiles through the Airtable interface for daily outreach work.

## Endpoint

<ParamField path="method" type="string" default="POST">
  POST
</ParamField>

<ParamField path="endpoint" type="string">
  `/api/airtable-sync/{campaign_id}`
</ParamField>

## Path Parameters

<ParamField path="campaign_id" type="string" required>
  The UUID of the campaign to sync to Airtable
</ParamField>

## Request

No request body is required. The campaign_id identifies which campaign's profiles to sync.

### Request Example

```bash
POST /api/airtable-sync/a1b2c3d4-e5f6-7890-abcd-ef1234567890
```

<Note>
This endpoint should only be called after successfully distributing profiles using the `/api/distribute/{campaign_id}` endpoint.
</Note>

<Warning>
Ensure your Airtable base has 80 tables named "Table 1" through "Table 80" before running this sync.
</Warning>

## Response

<ResponseField name="success" type="boolean" required>
  Indicates whether the Airtable sync completed successfully
</ResponseField>

<ResponseField name="campaign_id" type="string" required>
  UUID of the campaign that was synced
</ResponseField>

<ResponseField name="tables_synced" type="number" required>
  Number of Airtable tables updated (always 80)
</ResponseField>

<ResponseField name="total_records" type="number" required>
  Total number of records created in Airtable (always 14,400)
</ResponseField>

### Success Response Example

```json
{
  "success": true,
  "campaign_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "tables_synced": 80,
  "total_records": 14400
}
```

### Error Response Example

```json
{
  "success": false,
  "error": "Airtable API rate limit exceeded",
  "tables_synced": 45,
  "tables_remaining": 35
}
```

Each profile is synced to Airtable with the following fields:

```json

  "fields": {

    "Campaign Date": "2025-10-06",

    "Assigned At": "2025-10-06T14:30:00Z"
```

## Workflow Integration

Airtable Sync is the final step in the campaign creation workflow:

1. **Daily Selection (0-33%)** - ✅ Campaign created
2. **Distribution (33-66%)** - ✅ Distributed to VA tables
3. **Airtable Sync (66-100%)** - 📍 Current step

<CodeGroup>
```typescript Frontend Implementation
const handleAirtableSync = async (campaignId: string) => {
  try {
    setProgress(70);
    setProgressMessage('Syncing to Airtable...');
    
    const syncRes = await fetch(
      `${process.env.NEXT_PUBLIC_API_URL}/api/airtable-sync/${campaignId}`,
      { method: 'POST' }
    );
    
    const syncData = await syncRes.json();

    }
    
    console.log(`Synced ${syncData.total_records} records to ${syncData.tables_synced} tables`);
    setProgress(100);
    setProgressMessage('Campaign complete!');
    
    // Update campaign status to 'success'
    await updateCampaignStatus(campaignId, 'success');
    
  } catch (error) {
    console.error('Airtable sync failed:', error);
    await updateCampaignStatus(campaignId, 'failed');
  }
};
```

## Sync Process

The sync process follows these steps:

1. **Fetch Assignments** - Retrieve all 14,400 assignments from `daily_assignments` table
2. **Group by VA Table** - Organize profiles by VA table number (1-80)
3. **Batch Upload** - Upload 180 profiles to each Airtable table
4. **Rate Limiting** - Add delays between batches to respect Airtable API limits
5. **Error Handling** - Retry failed uploads up to 3 times

```python
# Simplified sync logic
for va_table in range(1, 81):
    profiles = get_profiles_for_table(campaign_id, va_table)
    
    # Batch upload to Airtable
    airtable_table = f"Table {va_table}"
    for batch in chunks(profiles, 10):  # Airtable allows 10 records per request
        airtable.create_records(airtable_table, batch)
        time.sleep(0.2)  # Rate limiting
```

## Performance Considerations

<Warning>
Airtable sync is the slowest step in the campaign workflow due to API rate limits.
</Warning>

**Timing breakdown:**
- **Per table:** 180 records ÷ 10 records/request = 18 requests
- **Per request:** ~200ms (including rate limiting)
- **Per table total:** ~3.6 seconds
- **All 80 tables:** ~5 minutes

**Optimization strategies:**
- Batch uploads (10 records per request - Airtable maximum)
- Parallel table uploads (up to 5 tables simultaneously)
- Exponential backoff for rate limit errors
- Connection pooling for API requests

### Airtable Table Structure

Each of the 80 tables must have these fields:

| Field Name | Type | Description |
|------------|------|-------------|
| Username | Single line text | Instagram username |
| Profile URL | URL | Instagram profile link |
| Campaign Date | Date | Date profiles were assigned |
| Status | Single select | Pending, In Progress, Completed |
| Assigned At | Date & time | Timestamp of assignment |

## Error Handling

| Error | Cause | Solution |
|-------|-------|----------|
| Invalid API key | `AIRTABLE_API_KEY` is incorrect or expired | Verify API key in Airtable account settings |
| Base not found | `AIRTABLE_BASE_ID` is incorrect | Check base ID in Airtable URL |
| Table not found | Missing VA tables in base | Create tables "Table 1" through "Table 80" |
| Rate limit exceeded | Too many requests to Airtable API | Sync will automatically retry with backoff |
| Field validation error | Airtable table structure doesn't match | Verify all required fields exist in tables |

## Rate Limiting

Airtable API limits:
- **Requests per second:** 5 requests/second
- **Records per request:** 10 records maximum
- **Total requests:** ~1,440 requests for full sync (80 tables × 18 requests)

**Built-in rate limiting:**
```python
import time

def upload_batch(table_name, records):
    try:
        airtable.batch_create(table_name, records)
    except RateLimitError:
        time.sleep(30)  # Wait 30 seconds
        upload_batch(table_name, records)  # Retry
```

2. **Verify field data:**
   - All usernames populated
   - Profile URLs correctly formatted
   - Campaign date matches current date
   - Status set to "Pending"

   ```sql
   -- Should return 0 rows
   ```
