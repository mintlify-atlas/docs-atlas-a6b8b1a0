## Installation

<Steps>
  <Step title="Navigate to client directory">
    ```bash
    cd client
    ```
  </Step>

  <Step title="Install dependencies">
    ```bash
    npm install
    ```
    
    This will install all required packages including Next.js, Supabase client, and UI components.
  </Step>

  <Step title="Set up environment variables">
    Copy the example environment file and configure your credentials:
    
    ```bash
    cp .env.example .env.local
    ```
    
    Edit `.env.local` with your actual credentials:
    
    ```bash .env.local
    # App URL
    NEXT_PUBLIC_APP_URL=http://localhost:3000
    
    # Backend API URL
    NEXT_PUBLIC_API_URL=http://localhost:5001
    
    # Supabase Configuration
    NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
    NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key-here
    
    # Authentication Credentials
    NEXT_PUBLIC_USERNAME=your-username
    NEXT_PUBLIC_PASSWORD=your-password
    NEXT_PUBLIC_OTP=123456
    
    # Campaign Configuration
    NEXT_PUBLIC_DAILY_SELECTION_TARGET=14400
    
    # Source Profiles Management
    NEXT_PUBLIC_DELETE_PASSWORD=delete123
    ```
    
    <Note>
      Get your Supabase URL and anon key from your [Supabase project dashboard](https://app.supabase.com).
    </Note>
  </Step>

  <Step title="Start the development server">
    ```bash
    npm run dev
    ```
    
    The application will start on `http://localhost:3000`.
  </Step>
</Steps>

## First Login

<Steps>
  <Step title="Navigate to login page">
    Open your browser and go to:
    ```
    http://localhost:3000/callum
    ```
  </Step>

  <Step title="Enter credentials">
    Use the credentials you configured in your `.env.local` file:
    
    - **Username**: Value from `NEXT_PUBLIC_USERNAME`
    - **Password**: Value from `NEXT_PUBLIC_PASSWORD`
    - **OTP**: Value from `NEXT_PUBLIC_OTP`
    
    <Note>
      The OTP is a static value from your environment variables, not a time-based code.
    </Note>
  </Step>

  <Step title="Access the dashboard">
    After successful login, you'll be redirected to the main dashboard at `/callum-dashboard`.
  </Step>
</Steps>

## Run Your First Scraping Job

<Steps>
  <Step title="Add source profiles">
    In the **Dependencies Card**, click **Edit Source Profiles**:
    
    1. Enter an Instagram username or URL (e.g., `@nike` or `https://instagram.com/nike`)
    2. Press Enter or click the **+** button
    3. The profile will be saved to your Supabase `source_profiles` table
    
    <Note>
      Source profiles are the Instagram accounts you want to scrape followers from. Add multiple accounts for better results.
    </Note>
  </Step>

  <Step title="Start scraping">
    Click the **Find Accounts** button to begin scraping:
    
    - **Scraping Phase (10-50%)**: Extracts followers from source accounts using Apify
    - **Ingesting Phase (60-80%)**: Saves profiles to database with gender detection
    - **Complete (100%)**: Shows success message
    
    <Warning>
      Scraping can take 5-15 minutes depending on the number of followers. The progress bar will update automatically.
    </Warning>
  </Step>

  <Step title="Create a campaign">
    Once scraping is complete, navigate to the **Payments Table** and click **Assign to VAs**:
    
    This will:
    1. Create a new campaign (0-33%)
    2. Select 14,400 unused profiles from your database
    3. Distribute profiles to 80 VA tables (33-66%)
    4. Sync to Airtable (66-100%)
    
    Each VA receives 180 profiles per day.
  </Step>

  <Step title="Monitor progress">
    Track your campaign in the **Campaigns Table**:
    
    - 🟢 **Green (Success)**: Campaign completed successfully
    - 🔴 **Red (Failed)**: Campaign encountered an error
    - ⚪ **Grey (Pending)**: Campaign is still processing
    
    <Note>
      You can view the **Username Status Card** to see how many unused profiles remain in your pool.
    </Note>
  </Step>
</Steps>

## Common Issues

<AccordionGroup>
  <Accordion title="Scraping returns no results">
    **Solution:**
    - Verify your Apify API key is valid in the backend `.env` file
    - Ensure source Instagram accounts are public
    - Check that your Apify actor has sufficient run quota
  </Accordion>

  <Accordion title="Cannot connect to Supabase">
    **Solution:**
    - Double-check your `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY`
    - Ensure your Supabase project is active (not paused)
    - Verify Row Level Security policies allow anonymous access
  </Accordion>

  <Accordion title="Login credentials not working">
    **Solution:**
    - Verify credentials in `.env.local` match exactly
    - Clear browser cache and cookies
    - Restart the development server with `npm run dev`
  </Accordion>
</AccordionGroup>
