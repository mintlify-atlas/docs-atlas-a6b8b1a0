## Platform-Specific Features

### Source Account Management

Configure Instagram accounts to scrape followers from:

<Steps>
  <Step title="Add Source Profiles">
    Navigate to the Dependencies Card and click "Edit Source Profiles"
  </Step>
  
  <Step title="Enter Account Details">
    Add Instagram usernames or URLs (e.g., `username` or `https://instagram.com/username`)
  </Step>
  
  <Step title="Validation">
    System automatically validates Instagram URLs and extracts usernames
  </Step>
  
  <Step title="Save to Database">
    Profiles are stored in the `source_profiles` table
  </Step>
</Steps>

### Gender-Based Filtering

Instagram scraping includes advanced gender detection:

<Note>
The system uses name-based detection algorithms to identify male profiles, which is the primary target for most campaigns.
</Note>

**Gender Distribution Tracking:**
- Male profiles (primary target)
- Female profiles (filtered out)
- Unknown (ambiguous names)

**Example Response:**
```json
{
  "genderDistribution": {
    "male": 450,
    "female": 400,
    "unknown": 150
  }
}
```

### Deduplication System

Instagram profiles are deduplicated using a two-table strategy:

1. **raw_scraped_profiles** - Stores all scraped data
2. **global_usernames** - Prevents duplicate assignments

<Warning>
Once a username is marked as `used`, it will not be assigned in future campaigns to prevent duplicate outreach.
</Warning>

## API Integration

### Instagram-Specific Endpoints

#### POST /api/scrape-followers

Scrapes Instagram follower lists with gender filtering:

```json
{
  "accounts": ["username1", "username2"],
  "targetGender": "male"
}
```

**Response:**
```json
{
  "success": true,
  "data": {
    "accounts": [...],
    "totalScraped": 1000,
    "totalFiltered": 450,
    "genderDistribution": {
      "male": 450,
      "female": 400,
      "unknown": 150
    }
  }
}
```

#### POST /api/ingest

Ingests Instagram profiles into the database:

```json
{
  "profiles": [
    {
      "id": "12345678",
      "username": "example_user",
      "full_name": "John Doe",
      "detected_gender": "male"
    }
  ]
}
```

## Limitations and Considerations

### Technical Limitations

<Warning>
**Rate Limiting**: Instagram imposes rate limits on scraping activities. The system includes built-in delays and retry logic to handle these limits gracefully.
</Warning>

<Note>
**Private Accounts**: The scraper can identify private accounts but cannot extract follower data from them. Ensure source accounts are public.
</Note>

## Troubleshooting

### Common Issues

<AccordionGroup>
  <Accordion title="Scraping Returns No Results">
    **Possible Causes:**
    - Source account is private
    - Instagram rate limiting active
    - Invalid account username
    - Apify API quota exceeded
    
    **Solutions:**
    - Verify account is public
    - Wait 24 hours and retry
    - Check username spelling
    - Review Apify dashboard for quota status
  </Accordion>
  
  <Accordion title="Gender Filter Not Working">
    **Possible Causes:**
    - Gender API key expired
    - Name data missing from profiles
    - Detection service unavailable
    
    **Solutions:**
    - Verify `GENDER_API_KEY` in environment
    - Check profile data includes `full_name`
    - Review backend logs for API errors
  </Accordion>
  
  <Accordion title="Duplicate Profiles Appearing">
    **Possible Causes:**
    - `global_usernames` table not updating
    - Deduplication logic skipped
    - Multiple campaigns running simultaneously
    
    **Solutions:**
    ```sql
    -- Check for duplicates
    SELECT username, COUNT(*) 
    FROM global_usernames 
    GROUP BY username 
    HAVING COUNT(*) > 1;
    ```
    
    - Ensure `used` flag is set correctly
    - Run campaigns sequentially, not in parallel
  </Accordion>
  
  <Accordion title="Insufficient Profiles for Campaign">
    **Possible Causes:**
    - Not enough unused profiles in database
    - Too many profiles marked as used
    - Scraping didn't complete successfully
    
    **Solutions:**
    ```sql
    -- Check available profiles
    SELECT COUNT(*) FROM global_usernames WHERE used = false;
    ```
    
    - Run additional scraping jobs
    - Consider running cleanup job to free old campaigns
    - Verify scraping completed (check progress reached 100%)
  </Accordion>
</AccordionGroup>
